# Documentation

This folder contains project documentation managed through spec-driven development (SDD).

## Structure

- `specs/` - Feature specifications linked to beads issues
- `history/` - Change log with auto-archiving

## Workflow

1. `/pm <feature>` - Create a new spec with beads issue
2. Refine the spec, add scenarios and requirements
3. Implement, tracking progress in beads
4. `/historian <summary>` - Record changes
5. `bd close <id>` - Mark complete
